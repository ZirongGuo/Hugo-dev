Set Up My Own Blog Using Github + Hugo

Welcome to my first blog!

Apr 05, 2026
2 minute read

0. Introduction

The tools we will used in this blog:

  • Github, for generating public sites, hosting images
  • Hugo, for gennerating static sites, applying blog themes
  • Picgo, for uploading images
  • VScode, for editing blogs in markdown format
  • VScode Extensions, for improving creative experience

Our approach to the construction is that we will learn how to set up a static site by Hugo at first, and then upload some of the key documents to Github.
Additionally, if you use a plain text editor, it will be quite challenging. So I will intersperse with explanations on how to use VScode + Picgo to optimize your creative experience.

1. Generate a static site

First, you should download Hugo.

Hugo download link: Hugo releases

Then, you should open the folder and excute the code below:

1

hugo new site dev

Now you should acquire a theme in Hugo:Hugo themes

Copy them to \dev:

Excute the code below:

1

hugo server -D

2. Markdown images: Github + Picgo

First, download Picgo: Picgo releases

You should create a Repository at Github. Then enter the configuration details.

3. Markdown Creative Enviroment Integration: VScode Extensions

  • Markdown All in One: Offers features such as auto-completion and key shortcut support, like Ctrl + B (bold), Ctrl + I (italic), Ctrl + K (insert link). When writing blogs, frequently using headings, lists, quotations, code blocks, etc., manually typing symbols is inefficient. Using this plugin can significantly improve the writing fluency.
  • Markdown Preview Enhanced: What you see is what you get. It offers real-time preview and export capabilities. You can open the dual-column real-time preview by pressing Ctrl + K V (of course, you can also view the changes in real time using Hugo, simply by saving the file).
